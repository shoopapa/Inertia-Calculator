Create App with Shared Data

This example shows how to use the Generate JavaScript using MATLAB Coder add-on to create a basic web application with a user interface that runs entirely in a web browser.

Create the App Project

1. In the MATLAB® current folder browser, navigate to a folder where you can create and store your app project and files.
2. Create the app project by entering this code in MATLAB command line. Specify a name and the directory for your app.
proj = webcoder.setup.project("My New App","Directory",pwd);
3. Open the index.html file as text. To open HTML files as text, right-click on the file in the folder browser and select Open as Text. To the HTML code, add six <para> fields and two <para> fields. This HTML code reflects these changes. The even <para> fields have id tag with labels "roll", "picth", and "yaw". The id tag is used to link the MATLAB code to the HTML.
<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta http-equiv="X-UA-Compatible" content="ie=edge">
<title>My New App</title>
</head>
<body>
<!-- Add your code here -->
<p>Roll:</p><p id="roll">0</p><br>
<p>Pitch:</p><p id="pitch">0</p><br>
<p>Yaw:</p><p id="yaw">0</p><br>
<!-- DO NOT REMOVE - This loads the JS/WASM files -->
<script async type="text/javascript" src="index.js"></script>
</body>
</html>
4. Create a new stored data class called StoreData.. This class enables data to be shared between callback functions and with the render function. Add the StoreData class to the project. For more information on static data, see Static Data.
classdef StoreData
methods (Access = private, Static)
function out = setgetAttitude(data)
persistent attitude
if isempty(attitude)
attitude = [NaN,NaN,NaN];
end
if nargin
attitude = data;
end
out = attitude;
end
end
methods (Static)
function setAttitude(data)
StoreData.setgetAttitude(data);
end
function data = getAttitude
data = StoreData.setgetAttitude();
end
end
end
5. Create a new function file called myOrientationCallback.m in the project folder.
function myOrientationCallback(Event)
webcoder.console.log('Recieved orientation event');
orientation = [Event.gamma,Event.beta,Event.alpha];
StoreData.setAttitude(orientation);
end
5. Add the function to the project clicking the Project tab > Add files button.
6. Add the MouseEvent label to the function by left-clicking on the function in the project and selecting Add label. In the Labels to Add dialog box, select Callbacks > DeviceOrientationEvent.
7. Open the start function and update the function as shown in this code. The addDeviceOrientationEventListener method registers the function, myOrientationCallback, to be executed when new sensor data arrives.
function start()
%START Run once at the start of the app
% The start function is run once at the start of the app to initialize
% values or perform other setup operations. The start function takes no
% inputs and no outputs.
webcoder.console.log('Starting...');
device = webcoder.sensor.DeviceMotion();
addDeviceOrientationEventListener(device,'myOrientationCallback');
end
8. Open the render function and update the function as shown in the following code. The current value in the StoreData class gets displayed to the screen into the <p> elements with ids equal to roll, pitch, and yaw every 1/60 second.
function render()
%RENDER Runs in an infinite loop
% The render function is run in an infinite loop at a specified update
% rate. Use the render function to make updates to graphics and dom
% objects. You can also use the render() function to perform simple
% polling of data. The render function takes zero inputs and zero
% outputs.
webcoder.console.log('... Updating the display ...');
attitude = StoreData.getAttitude();
rollElem = webcoder.dom.Element('roll');
rollElem.setInnerText(sprintf('%f',attitude(1)));
pitchElem = webcoder.dom.Element('pitch');
pitchElem.setInnerText(sprintf('%f',attitude(2)));
yawElem = webcoder.dom.Element('yaw');
yawElem.setInnerText(sprintf('%f',attitude(3)));
end

Build the App

To build the app project, enter this code at the command prompt. The amount of time it take for the app to build depends on your host computer.
proj = openProject(pwd);
webcoder.build.project(proj);
The resulting app is the set of 4 files located in the dist folder.

Run the App

From the MATLAB command line, you can launch the app in a local server. To create the server and launch the app in the dist folder, enter this code.
localServer = webcoder.utilities.DevelopmentServer("Directory",'dist',"Port",8125);
start(localServer);
Stop the server at any time using this command.
stop(localServer);
In a Chrome™ browser, you can open the app by navigating to the address http://localhost:8125 . After the page opens, open the browser developer tools by pressing Ctrl+Shift+i. In the top right corner of the developer tools window, expand the vertical ellipsis and click More tools > Sensors. In the Sensors pane, set Orientation to Custom orientation. Move the image of the phone to simulate the device sensors and generate a response in your app display.
